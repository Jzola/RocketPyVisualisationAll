Description

This site allows users to explore the existing UML for the RocketPyVRSimulation program. Main Scene Top Level The Main Scene holds all of the top level /parent components required to run the simulation

Conventions Used: -Parent GameObjects have an aggregation relationship with Child Objects. -Components use a composition relationship with the GameOBject they are placed on. -Game Objects have a composition relationship to their needed scripts (when used as components) -Scripts that depend on other scripts , or other GameObjects that they are not components of, have a dependency relationship

Not all objects can fit in the same diagram, so where relationships have been split, the root component is grey, and any leaf components that are expanded elsewhere are in yellow
